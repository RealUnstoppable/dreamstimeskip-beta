## 2025-02-28 - Avoid Stale Closure Variables When Refactoring to Global Event Listeners
**Learning:** When moving event listeners (like drag-and-drop `pointerup`) from inside a loop (where they capture a specific `idx` in closure) to the global `document` scope, you lose access to the correct array index. If you try to use a static variable or assume an uninitialized `dataset` attribute, it can silently break logic (like always dropping at index 0 because `parseInt(undefined)` is `NaN` which is coerced to `0` by `splice`).
**Action:** When extracting event listeners to the global scope, dynamically calculate the active element's index by querying the DOM directly in the handler (e.g., `Array.from(container.querySelectorAll('.item')).indexOf(activeItem)`) rather than relying on stale closure variables or assuming missing data attributes.
